# 在线构建APK - 最简单方法

## 方法一：使用GitHub Desktop（推荐）

### 准备工作：
1. 下载GitHub Desktop：https://desktop.github.com
2. 注册GitHub账号：https://github.com
3. 确保项目在：`C:\Users\Administrator\WorkBuddy\Claw\HushangAuntieInventory`

### 步骤1：上传项目到GitHub
1. 打开GitHub Desktop
2. 点击"Add an existing repository"
3. 选择项目目录：`C:\Users\Administrator\WorkBuddy\Claw\HushangAuntieInventory`
4. 点击"Publish repository"
5. 填写信息：
   - Name: `hushang-auntie-app`
   - Description: 沪上阿姨奶茶店管理系统
   - 保持"Public"选中
   - 不要勾选"Initialize with README"
6. 点击"Publish repository"

### 步骤2：使用Microsoft App Center构建
1. 访问：https://appcenter.ms
2. 点击"Sign up"注册（可以使用GitHub账号登录）
3. 登录后点击"Add new" → "Add new app"
4. 填写应用信息：
   - App name: `HushangAuntie`
   - Release type: `Production`
   - OS: `Android`
   - Platform: `React Native`
5. 点击"Add new app"

### 步骤3：连接GitHub仓库
1. 在App Center中，进入您的应用
2. 点击"Build" → "Connect to repository"
3. 选择GitHub，授权访问
4. 选择您的仓库：`hushang-auntie-app`
5. 选择分支：`main`

### 步骤4：配置构建
1. 点击"Configure build"
2. 选择Node版本：`18.x`
3. 构建脚本会自动检测
4. 点击"Save & Build"

### 步骤5：下载APK
1. 等待构建完成（约10-15分钟）
2. 构建成功后，点击"Download"
3. 下载APK文件到电脑

## 方法二：使用Expo在线构建

### 步骤1：安装Expo CLI
```bash
npm install -g expo-cli
```

### 步骤2：登录Expo
```bash
expo login
```
- 如果没有账号，在浏览器中注册

### 步骤3：构建APK
```bash
cd C:\Users\Administrator\WorkBuddy\Claw\HushangAuntieInventory
expo build:android
```

### 步骤4：选择构建类型
- 选择：`APK`
- 等待构建完成（约20-30分钟）
- 获取下载链接

## 方法三：使用GitLab CI/CD

### 步骤1：创建GitLab账号
1. 访问：https://gitlab.com
2. 注册账号

### 步骤2：创建新项目
1. 点击"New project"
2. 选择"Import project" → "Repository by URL"
3. 输入GitHub仓库URL
4. 点击"Create project"

### 步骤3：自动构建
- GitLab会自动检测并开始构建
- 等待构建完成
- 下载APK文件

## 📱 安装APK到手机

### 方法A：USB传输
1. 用USB数据线连接手机和电脑
2. 将APK文件复制到手机存储
3. 在手机上打开文件管理器
4. 找到APK文件并点击安装
5. 允许"安装未知来源应用"

### 方法B：电子邮件
1. 将APK文件作为邮件附件发送到自己的邮箱
2. 在手机上打开邮件应用
3. 下载附件
4. 安装应用

### 方法C：网盘分享
1. 上传APK到百度网盘或Google Drive
2. 在手机上打开网盘应用
3. 下载文件
4. 安装应用

## 🔧 常见问题解决

### 问题1：构建失败
**可能原因**：
- 网络连接问题
- 依赖包缺失
- 配置错误

**解决方法**：
1. 检查构建日志中的具体错误
2. 确保所有依赖已正确安装
3. 重新尝试构建

### 问题2：GitHub仓库创建失败
**解决方法**：
1. 检查GitHub账号是否正常
2. 确保仓库名称唯一
3. 尝试不同的仓库名称

### 问题3：APK安装失败
**解决方法**：
1. 确保手机Android版本 >= 5.0
2. 开启"安装未知来源应用"权限
3. 重新下载APK文件
